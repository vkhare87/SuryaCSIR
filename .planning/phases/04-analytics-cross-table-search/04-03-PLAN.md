<objective>
Built the global search / command palette accessible from every authenticated page via Ctrl+K (or Cmd+K on macOS). The CommandPalette component performs client-side full-text search across all six DataContext tables (staff, projects, phDStudents, scientificOutputs, ipIntelligence, equipment) and displays up to 3 results per category in a modal overlay. Selecting a staff result navigates to the staff portfolio view. Layout.tsx was updated with a global keydown listener and the CommandPalette mount, plus mobile menu bug fixes.

Purpose: Let any authenticated user jump to any institute record from any page without navigating through menus — a universal entry point to the institute knowledge graph.
Output: CommandPalette.tsx (new component), updated Layout.tsx (Ctrl+K listener, CommandPalette mount, mobile menu fixes).
</objective>

<tasks>
## Tasks

### Task 1: CommandPalette component

- **type:** implementation
- **files:** `src/components/CommandPalette.tsx`
- **action:** Created default-export `CommandPalette` component accepting `{ isOpen, onClose }` props. Renders as a fixed full-screen overlay (z-50) with backdrop-blur when isOpen is true; returns null otherwise. Search input auto-focuses on open. Calls `searchAll(query, data)` — a pure function that runs six independent `.filter().slice(0, 3)` passes over the DataContext arrays: staff (Name, Designation, Division, CoreArea, Expertise); projects (ProjectName, ProjectNo, SponsorerName, PrincipalInvestigator); phDStudents (StudentName, Specialization, SupervisorName, ThesisTitle); scientificOutputs (title, journal, authors array); ipIntelligence (title, type, status, inventors array); equipment (Name, EndUse, Division, IndenterName, OperatorName). All matching is case-insensitive substring. Results are grouped into labelled sections (ResultSection) with icon-accented rows (ResultRow). Staff results navigate to /staff/:id (portfolio view). Other results navigate to their respective list pages (/projects/:id, /phd, /intelligence, /facilities). Empty query state shows a quick-navigate icon grid for the six main sections. Escape key closes and clears. `SEARCH-04` scoping: search operates over DataContext data, which is already filtered by the user's role at the context level — no additional role filter needed in CommandPalette itself. Footer shows keyboard hint legend (↑↓ / Enter / Esc).
- **verify:** Open the app, press Ctrl+K — overlay appears with autofocused input and quick-navigate grid. Type a partial staff name — Staff section appears with up to 3 matches. Click a staff result — navigates to /staff/:id and palette closes. Press Escape — palette closes and query is cleared.
- **acceptance_criteria:** Default export. Accepts isOpen and onClose props. Renders null when isOpen is false. Staff results navigate to /staff/:id (not to /staff list). `RESULT_LIMIT = 3` constant caps each category. No direct Supabase calls — reads from useData().

### Task 2: Layout.tsx — Ctrl+K listener and CommandPalette mount

- **type:** implementation
- **files:** `src/components/layout/Layout.tsx`
- **action:** Added `commandPaletteOpen` boolean state. Added a `useEffect` on mount that attaches a window keydown listener: fires `setCommandPaletteOpen(true)` when `(e.metaKey || e.ctrlKey) && e.key === 'k'`, with `e.preventDefault()` to suppress browser default. Listener is removed on cleanup. Added a Search icon button in the top header bar that also opens the palette on click. Mounted `<CommandPalette isOpen={commandPaletteOpen} onClose={() => setCommandPaletteOpen(false)} />` at the bottom of the Layout return, outside the main content area (renders at root z-50 layer). Also fixed a mobile menu bug: the mobile drawer close button now calls `setMobileMenuOpen(false)` correctly and the mobile nav links close the drawer on navigate.
- **verify:** Ctrl+K opens the palette from any authenticated page (Dashboard, Staff, Projects, etc.). Clicking the Search icon in the nav header opens the palette. The palette is not accessible from the login/setup pages (those routes are outside Layout).
- **acceptance_criteria:** Ctrl+K works on Windows/Linux; Cmd+K works on macOS (both ctrlKey and metaKey handled). The palette mounts once at the Layout level — not re-created per route. Mobile menu closes on nav item click.
