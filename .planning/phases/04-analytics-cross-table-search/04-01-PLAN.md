<objective>
Built division scorecards and charts for Director and Division Head dashboards. Introduced a shared KpiCard component, an analytics utility (getDivisionMetrics / personNamesMatch / normalizePersonName) that aggregates cross-table metrics per division, and wired both dashboard views to render live KPI cards and a Recharts BarChart comparing divisions on project count and scientific output count. The Director gets an institute-wide view with all-division scorecards and a comparison chart; the Division Head gets a role-scoped view driven by divisionCode from AuthContext, with staff and project detail tables beneath the KPI row.

Purpose: Give leadership a single screen that answers "how is each division performing?" without needing to cross-reference separate list pages.
Output: KpiCard.tsx (shared primitive), analytics.ts (getDivisionMetrics, personNamesMatch, normalizePersonName), updated DirectorView.tsx and DivisionHeadView.tsx.
</objective>

## Tasks

### Task 1: Analytics utility — getDivisionMetrics

- **type:** implementation
- **files:** `src/utils/analytics.ts`
- **action:** Created `DivisionMetric` and `StaffPortfolio` interfaces (both exported). Added `normalizePersonName` (strips honorifics, lowercases, collapses whitespace) and `personNamesMatch` (bidirectional substring match after normalization) for fuzzy cross-table name matching. Added `getDivisionMetrics` which accepts all six DataContext arrays and returns one `DivisionMetric` per division: staffCount (staff.Division === divCode), activeProjectCount and projectCount (projects.DivisionCode), phdStudentCount (PhDs whose supervisor or co-supervisor name fuzzy-matches any division staff member name), scientificOutputCount (scientificOutputs.divisionCode), and equipmentCount (equipment.Division).
- **verify:** `npm run build` exits 0 and `npm run lint` reports 0 errors; `getDivisionMetrics` is importable from `src/utils/analytics`.
- **acceptance_criteria:** Function returns typed DivisionMetric[] with no TypeScript errors; personNamesMatch handles null/undefined without throwing.

### Task 2: Shared KpiCard component

- **type:** implementation
- **files:** `src/components/ui/KpiCard.tsx`
- **action:** Created named export `KpiCard` with props `{ label, value, icon?, sublabel? }`. Renders a card using semantic Tailwind tokens (bg-surface, border-border, text-terracotta, text-near-black). Value renders at 3xl serif; label at 11px uppercase tracking-widest in muted grey. Moved to src/components/ui/ per CLAUDE.md convention (shared UI primitive). All 11 dashboard views import from '../../components/ui/KpiCard'.
- **verify:** All dashboard views import from correct path; no ./KpiCard imports remain.
- **acceptance_criteria:** Card renders label, numeric value, optional icon, and optional sublabel. Uses semantic token classes (bg-surface, border-border, text-terracotta, text-near-black).

### Task 3: DirectorView and DivisionHeadView dashboards

- **type:** implementation
- **files:** `src/pages/dashboards/DirectorView.tsx`, `src/pages/dashboards/DivisionHeadView.tsx`
- **action:** DirectorView: Calls `getDivisionMetrics` with all DataContext arrays (memoized). Renders five institute-wide KpiCards (Total Staff, Active Projects, PhD Students, Equipment, Scientific Outputs). Below that, renders per-division score cards in a responsive grid showing staffCount, activeProjectCount, scientificOutputCount, phdStudentCount. Then a Recharts ResponsiveContainer BarChart with two Bar series (projectCount terracotta #c96442, scientificOutputCount warm-grey #5e5d59) comparing all divisions by divCode on the X-axis. Finally a Division Breakdown table from the raw `divisions` array showing current/sanctioned strength and HoD. DivisionHeadView: Reads divisionCode from AuthContext. Renders five role-scoped KpiCards (Division Staff, Active Projects, PhD Supervisees, Outputs, Equipment) pulling from DataContext arrays (already filtered at context level by role). Renders two side-by-side tables — Staff (Name, Designation, Group) and Projects (ProjectName, status badge, PrincipalInvestigator).
- **verify:** Log in as Director and navigate to /dashboard — five KPI cards appear with non-zero values, division scorecards grid renders for each division, BarChart renders with two bar series. Log in as DivisionHead — five KPI cards appear, staff and project tables render.
- **acceptance_criteria:** BarChart uses fill="#c96442" and fill="#5e5d59". No raw `bg-white` or `text-gray-*` classes — palette uses hex tokens. Data-empty states render without JS errors.
