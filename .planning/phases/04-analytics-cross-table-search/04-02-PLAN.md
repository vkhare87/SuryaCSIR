<objective>
Built the Scientist Portfolio View — a cross-table staff detail page at route /staff/:id. The page aggregates six DataContext tables (projects, projectStaff, phDStudents, scientificOutputs, ipIntelligence, equipment) into a single unified profile view for any staff member. It also introduced career date helpers in dateUtils.ts (getRetirementDate, getAgeFromDOB, getServiceYears, getYearsInGrade) and the getStaffPortfolio aggregation function in analytics.ts.

Purpose: Let scientists, admins, and directors inspect any staff member's complete research footprint from a single page rather than hunting across six list pages.
Output: Updated StaffDetail.tsx (full portfolio page), getStaffPortfolio in analytics.ts, four new career date helpers in dateUtils.ts.
</objective>

## Tasks

### Task 1: Career date helpers in dateUtils.ts

- **type:** implementation
- **files:** `src/utils/dateUtils.ts`
- **action:** Added four JSDoc-annotated utility functions alongside the existing parseDate/diffInDays/formatDate functions. `getRetirementDate(dob)` — parses DOB string, returns a Date at DOB + 60 years (CSIR retirement age), or null if unparseable. `getAgeFromDOB(dob)` — returns whole years from DOB to today using diffInDays / 365.25, or null. `getServiceYears(doj)` — returns whole years from DOJ (date of joining) to today, or null. `getYearsInGrade(doapp)` — returns whole years from DoAPP (date of appointment to current post) to today, or null. All four follow the existing parseDate-then-calculate pattern in the file.
- **verify:** `npm run build` exits 0; StaffDetail.tsx imports all four without TypeScript errors.
- **acceptance_criteria:** Functions handle null/undefined input gracefully (return null, not throw). Return type is `number | null` or `Date | null` as appropriate.

### Task 2: getStaffPortfolio aggregation in analytics.ts

- **type:** implementation
- **files:** `src/utils/analytics.ts`
- **action:** Added `getStaffPortfolio` function accepting a params object of all eight DataContext arrays plus staffId. Returns `StaffPortfolio | null` (null when staffId not found in staff array). Portfolio assembly: finds the StaffMember by ID; uses personNamesMatch for fuzzy name matching throughout. projectAssignments = projectStaff rows where StaffName fuzzy-matches member.Name. supervisedPhDs = phDStudents where SupervisorName matches. coSupervisedPhDs = phDStudents where CoSupervisorName matches. linkedProjects = projects where PrincipalInvestigator matches OR ProjectNo appears in any of the assignment/PhD project sets. publications = scientificOutputs where any author matches. ipAssets = ipIntelligence where any inventor matches. assignedEquipment = equipment where IndenterName or OperatorName matches. Returns the StaffPortfolio object with all seven derived arrays.
- **verify:** StaffDetail.tsx calls getStaffPortfolio and renders portfolio data without TypeScript errors.
- **acceptance_criteria:** getStaffPortfolio returns null cleanly for an unknown staffId (no exception). All seven derived arrays are populated using personNamesMatch, so honorific variations (Dr., Prof., Sh.) resolve correctly.

### Task 3: StaffDetail portfolio page

- **type:** implementation
- **files:** `src/pages/StaffDetail.tsx`
- **action:** Rewrote /staff/:id as a full cross-table portfolio page. Layout is a 3-column grid on large screens: left column (1/3) holds the ID Card (avatar initial, name, designation, badges for Group/Level/Category, contact info, Vidwan link) and Career Intelligence card (2x2 grid showing Age / Service Years / Years in Grade / Retirement Date with a rose-tinted retirement-soon alert when ≤365 days remain). Right column (2/3) holds: a 5-cell stat bar (Active Projects, Publications, PhD Students, IP Assets, Equipment); Research & Expertise (CoreArea, Expertise); Organisational Hierarchy (reports-to link + direct reports list, both as clickable /staff/:id links); Linked Projects list (project badge + link to /projects/:id); PhD Mentorship section (combined supervisor + co-supervisor list with status badges and thesis titles); Publications list (title, journal/year, impact factor badge, citation badge, DOI link); IP Portfolio (title, inventors, type badge, status badge, filing/grant dates); Assigned Equipment (name, location, working status badge, indenter/operator labels). Each section renders conditionally — empty sections (zero records) are suppressed rather than showing empty cards. Not-found state shows a message and a back button.
- **verify:** Navigate to /staff/:id for a valid staff ID — portfolio page renders with all populated sections. Navigate to /staff/nonexistent — "Staff member not found" message and Go Back button appear. Clicking a direct report or reporting manager navigates to their portfolio. Clicking a linked project navigates to /projects/:id.
- **acceptance_criteria:** Page imports from `../utils/analytics` (getStaffPortfolio) and `../utils/dateUtils` (date helpers). No direct Supabase calls — all data from useData(). Retirement-soon highlight renders in rose tones when daysToRetirement is between 1 and 365.
